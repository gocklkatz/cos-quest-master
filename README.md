# COS Quest Master

An AI-powered, gamified learning app for [InterSystems ObjectScript](https://docs.intersystems.com/irislatest/csp/docbook/DocBook.UI.Page.cls?KEY=GCOS_intro). Write real ObjectScript in a browser-based code editor, execute it on a live IRIS instance, and let Claude evaluate your code and guide you through progressive quests.

## Overview

Quest Master teaches ObjectScript through a quest-driven loop:

1. **Receive a quest** — generated by the Claude API, tailored to your current level
2. **Write code** — in a Monaco editor directly in the browser
3. **Execute on IRIS** — code is sent to a real IRIS Docker instance via REST, compiled and run
4. **Get evaluated** — output is sent back to Claude for feedback and scoring
5. **Earn XP** — level up, see animated XP gains, and unlock the next quest

**Features:**
- Monaco editor with ObjectScript syntax
- Live IRIS connection indicator with health polling
- Quest system with hard-coded starter quests and AI-generated follow-ups
- XP and level progression (Apprentice → Journeyman → Master)
- Animated XP popups on quest completion
- Skill tree: unlock branches as you level up
- Quest log: full history of completed quests
- Claude AI integration for quest generation and code evaluation (requires Anthropic API key; gracefully disabled without one)
- Settings modal for IRIS connection and API key configuration

## Architecture

```
┌─────────────────────────────────────────┐
│           Browser (Angular App)         │
│                                         │
│  ┌─────────────┐  ┌──────────────────┐  │
│  │ Quest UI    │  │ Monaco Editor    │  │
│  │ XP/Level    │  │ (ObjectScript)   │  │
│  └──────┬──────┘  └────────┬─────────┘  │
│         │                  │            │
│         ▼                  ▼            │
│   Claude API         IRIS REST API      │
│   (quest gen,        /api/quest/        │
│    evaluate)         execute/compile    │
└─────────────────────────────────────────┘
         │                  │
         ▼                  ▼
   api.anthropic.com   localhost:52773
                       (IRIS in Docker)
```

There is no separate backend server. All API calls are made directly from the browser. The user supplies their own Anthropic API key (stored in `localStorage`) and their IRIS connection details via the settings panel.

## IRIS Setup

### One-command setup

```bash
cd quest-master/iris
./setup.sh
```

This script:
1. Pulls `containers.intersystems.com/intersystems/iris-community:2025.1`
2. Starts the container via Docker Compose (bind-mount for data persistence)
3. Waits for the container to become healthy
4. Sets the `_SYSTEM` password
5. Deploys and compiles `QuestMaster.REST.Execute`
6. Registers the `/api/quest/` web application
7. Verifies the health endpoint

Optional arguments: `./setup.sh [container-name] [password]` (defaults: `my-iris` / `password`).

### Verify

```bash
curl -u _SYSTEM:password http://localhost:52773/api/quest/health
# → {"status":"ok","namespace":"USER"}
```

### Redeploying after changes to the REST class

```bash
docker cp quest-master/iris/QuestMaster.REST.Execute.cls my-iris:/tmp/
docker exec -i my-iris bash -c 'iris session IRIS -U USER' <<'EOF'
Do ##class(%SYSTEM.OBJ).Load("/tmp/QuestMaster.REST.Execute.cls","ck-d")
Halt
EOF
```

## Frontend Setup

```bash
cd quest-master
npm install

```

Open `http://localhost:4200` in your browser. The dev server proxies `/api/quest/*` and `/api/atelier/*` to `http://localhost:52773` automatically.

### First-time configuration

Click the settings icon in the header and fill in:

- **IRIS URL** — `http://localhost:52773` (default)
- **Namespace** — `USER`
- **Username / Password** — your IRIS credentials (default: `_SYSTEM` / `SYS` or `password`)
- **Anthropic API Key** — your `sk-ant-…` key (stored in `localStorage` only, never sent to any server other than `api.anthropic.com`)

## IRIS REST API Reference

Base path: `/api/quest/`

| Method | Path | Description |
|---|---|---|
| `GET` | `/health` | Returns `{"status":"ok","namespace":"USER"}` |
| `POST` | `/execute` | Executes ObjectScript code, returns captured output |
| `POST` | `/compile` | Compiles an ObjectScript class from source |

### POST /execute

```json
// Request
{ "code": "Write \"Hello, World!\", !" }

// Response (success)
{ "success": 1, "output": "Hello, World!\n" }

// Response (error)
{ "success": 0, "error": "ERROR (line 1): <DIVIDE>" }
```

### POST /compile

```json
// Request
{ "className": "My.Class", "source": "Class My.Class { ... }" }

// Response
{ "success": 1 }
```

## Development Notes

### ObjectScript execution model

`XECUTE` only accepts single-line strings. The `/execute` endpoint splits multi-line code on `$C(10)` and executes each line separately. Variables persist across lines within the same request (same process context).

### IO capture

Output is captured by redirecting `$IO` to a temp file (`WNS` mode) before `XECUTE`, then read back (`RS` mode) after execution. This works reliably in the REST/CSP context where `$IO` is the TCP superserver socket.

### Running the ObjectScript unit tests

```bash
docker exec -i my-iris bash -c 'iris session IRIS -U USER' <<'EOF'
Set ^UnitTestRoot = "/tmp/tests/"
Do ##class(%UnitTest.Manager).RunTest("Test.RESTDispatch","/nodelete/noload")
Halt
EOF
```

## Security Notes

- The Anthropic API key is stored **only in your browser's `localStorage`** and sent directly to `api.anthropic.com`. It is never proxied through IRIS or any other server.
- IRIS credentials are stored in `localStorage` and used only for Basic Auth headers to your local IRIS instance.
- This project is designed for **local development only**. Do not expose your IRIS instance to the public internet without proper authentication and hardening.
